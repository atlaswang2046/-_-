# Note规范模板

## 1. 目的

用于沉淀当前原型中 `note` 的统一规范，记录：

- 页面位置（画布坐标）
- note 位置（画布坐标）
- note 样式与**5 维度结构化内容**
- 页面与 note 的一一对应关系

**基线文件**：`原型图/客户营养餐.pen`

## 2. 当前画布布局（实测）

### 2.1 页面网格

- **页面统一尺寸**：`1920 × 1080`
- **页面行距（纵向）**：`1220`
- **主列步进（横向）**：`2300`
- **右侧 note 偏移**：`1940`
- **当前实际页面列坐标示例**：`x = 0 / 2300 / 4600 / 6900 / 9460`
- **当前实际页面行坐标示例**：`y = 0 / 1220 / 2440 / 3660 / 4880 / 6430 / 7650 / 9009`

### 2.2 Note布局

- **位置策略**：每个页面右侧一个 note，一一对应。
- **标准宽度**：`250`
- **标准高度**：按内容自适应，建议控制在 `220-260` 之间。
- **水平关系**：`note.x = page.x + 1940`
- **垂直关系**：`note.y = page.y`（与页面顶部对齐）
- **联动规则**：若后续页面右移，note 也必须同步跟随新的页面坐标重新计算。禁止把 note 放在页面下方或压盖其它页面内容。

## 4. Note样式规范（当前）

### 4.1 节点类型

- **类型**：原生 `note`。
- 无子节点，正文为单字段 `content`。

### 4.2 视觉样式

- **底色**：浅米黄色（系统 note 样式）。
- **顶部**：`Note` 标题与下拉箭头。
- **分隔**：头部下方浅棕色虚线。
- **文本**：深棕色，左对齐，连续段落。引入 Emoji 进行结构化物理隔离，便于机器提取特征。

### 4.3 尺寸规范

- **宽度**：统一保持 `250`，不再使用短宽 note。
- **高度**：自适应文本（当前主要在 `220-280`）。内容较多的配置页允许到 `300-520`，但绝对底线是不能压盖到下方的物理页面。

## 5. 内容模板规范

为了确保 Codex 能精准解析业务逻辑并映射 Element Plus 组件，正文内容必须严格遵循 **5 维度结构**。

- **标题格式**：`📋 {页面编号} {页面名}`
- **命名统一**：`note{页面编号}`，如 `note2_1_1`。
- **内容要求**：文本控制为“短句”，必须使用键值对结构，抛弃大段口语化描述。
- **5 维度字段及前缀符号（必填项）**：
  - `🔵 业务逻辑`：数据来源、字段业务含义及去向。
  - `🟢 操作流程`：触发条件与产生的明确结果（跳转、刷新、回填等）。
  - `🟠 必填项`：明确标识（是/否）。
  - `🔴 校验规则`：格式要求、数值边界及异常提示文案。
  - `🟡 约束项`：前置条件、状态卡点及角色权限限制。

## 6. 更新流程（执行模板）

1. 先读取页面节点坐标（`snapshot_layout maxDepth=0`）。
2. 读取全部 note 节点（`batch_get patterns:[{type:"note"}]`）。
3. 比对页面名与 note 标题，修正旧编号或旧页面名。
4. 按关系批量更新：`note.x = page.x + 1940`、`note.y = page.y`。
5. 宽度统一设定至 `250`，高度按内容检查安全范围。
6. 检查重复 note：若同一页面存在多个 note，合并 5 维度信息后，仅保留 1 个最终节点。
7. 执行重叠检查（`snapshot_layout problemsOnly=true`）。

## 7. 验收清单

- [ ] 每个页面都有且只有一个对应 note。
- [ ] note 在页面右侧，不压盖页面内容。
- [ ] 同行页面的 note 视觉对齐。
- [ ] note 内容严格包含 🔵、🟢、🟠、🔴、🟡 5 个维度标识。
- [ ] 页面右移或新增页面后，note 坐标同步更新。
- [ ] 页面重命名后，note 标题与 name 字段同步更新。
