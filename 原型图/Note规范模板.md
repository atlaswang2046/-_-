# Note规范模板（客户营养餐）

## 1. 目的

用于沉淀当前原型中 `note` 的统一规范，记录：

- 页面位置（画布坐标）
- note 位置（画布坐标）
- note 样式与内容结构
- 页面与 note 的一一对应关系

基线文件：`原型图/客户营养餐.pen`

## 2. 当前画布布局（实测）

### 2.1 页面网格

- 页面统一尺寸：`1920 × 1080`
- 页面行距（纵向）：`1220`
- 主列步进（横向）：`2300`
- 右侧 note 偏移：`1940`
- 当前实际页面列坐标示例：
  - `x = 0 / 2300 / 4600 / 6900 / 9460`
- 当前实际页面行坐标示例：
  - `y = 0 / 1220 / 2440 / 3660 / 4880 / 6430 / 7650 / 9009`

### 2.2 Note布局

- 位置策略：每个页面右侧一个 note，一一对应
- note 标准宽度：`250`
- note 高度：按内容自适应，建议控制在 `220-260`
- note 与页面水平关系：`note.x = page.x + 1940`
- note 与页面垂直关系：`note.y = page.y`（与页面顶部对齐）
- 若后续页面右移，note 也必须同步跟随新的页面坐标重新计算
- 禁止把 note 放在页面下方或压住其它页面

## 4. Note样式规范（当前）

### 4.1 节点类型

- 类型：原生 `note`
- 无子节点，正文为单字段 `content`

### 4.2 视觉样式

- 底色：浅米黄色（系统 note 样式）
- 顶部：`Note` 标题与下拉箭头
- 分隔：头部下方浅棕色虚线
- 文本：深棕色，左对齐，连续段落

### 4.3 尺寸规范

- 推荐宽度：`250`
- 高度：自适应文本（当前主要在 `220-280`）
- 内容较多的配置页允许到 `300-520`，但必须避免压到下方页面
- 宽度统一保持 `250`，不再使用短宽 note

## 5. 内容模板规范

- 标题：`📋 {页面编号} {页面名}`
- 主体建议 3 段，优先保留：
  - `【核心逻辑】`
  - `【主操作】`
  - `【关键校验】`
- 文本控制为“短句+分段”，避免过长导致 note 高度过大
- 命名统一为：`note{页面编号}`，例如 `note5_3_1`

示例：

```text
📋 1.0 院区基础设施

【核心逻辑】院区主数据列表，支持筛选、启停、模板复制。
【主操作】新增院区(跳1.1)、编辑(跳1.1)、详情(抽屉)、用户管理(跳1.2)。
【关键校验】院区编码唯一；启停前校验关联数据。
```

## 6. 更新流程（执行模板）

1. 先读取页面节点坐标（`snapshot_layout maxDepth=0`）。
2. 读取全部 note 节点（`batch_get patterns:[{type:\"note\"}]`）。
3. 比对页面名与 note 标题，修正旧编号或旧页面名。
4. 按关系批量更新：`note.x = page.x + 1940`、`note.y = page.y`。
5. 宽度统一到 `250`，高度按内容控制在安全范围内。
6. 检查重复 note，若同一页面存在多个 note，仅保留 1 个最终版本。
7. 执行重叠检查（`snapshot_layout problemsOnly=true`）。

## 7. 验收清单

- 每个页面都有且只有一个对应 note。
- note 在页面右侧，不压盖页面内容。
- 同行页面的 note 视觉对齐。
- note 内容段落简洁，可读性明确。
- 页面右移或新增页面后，note 坐标同步更新。
- 页面重命名后，note 标题与 name 字段同步更新。
