## 页面功能一句话（一级+二级）

- `1.0` 工作台总览 - Element Plus：汇总今日经营与待办，提供高频业务入口。
- `2.0` 调度日历全屏页：以日历方式查看资源占用并处理排期冲突。
- `3.0` 全渠道订单信息处理：统一承接多渠道预约并执行审核、详情、改期等处理。
- `4.0` 全渠道订单处理池：集中处理履约中的核销、退改与退款流程。
- `5.0` 团队预订与批量导入：管理团队订单全流程及批量导入相关操作。
- `0.1` 全局 + 全局帮助抽屉：提供跨页面操作指引、FAQ 与教程入口。
- `2.1` 调度日历全屏页 + 周视图：按周维度查看预约分布并快速定位排期。
- `2.2` 调度日历全屏页 + 月视图：按月维度查看预约密度与资源占用趋势。
- `2.3` 调度日历全屏页 + 日历交互注释：说明日历核心交互规则与状态含义。
- `3.1` 全渠道订单信息处理 + 客户信息：录入或匹配客户档案，开启建单流程。
- `3.2` 全渠道订单信息处理 + 预订房间：完成入住离店与房型房量选择并校验库存。
- `3.3` 全渠道订单信息处理 + 预订项目：配置康养项目与时段并校验资源可用性。
- `3.4` 全渠道订单信息处理 + 费用确认：汇总房费、项目费与附加费形成应收金额。
- `3.5` 全渠道订单信息处理 + 预约确认：核对订单全量信息并发起预占确认。
- `3.6` 全渠道订单信息处理 + 到店收款确认页面：对预占订单执行到店收款与到账确认。
- `4.1` 全渠道订单处理池 + 核销确认弹窗：确认服务已履约并记录核销结果。
- `4.2` 全渠道订单处理池 + 退改申请弹窗：发起退改申请并校验可退改规则。
- `4.3` 全渠道订单处理池 + 独立部分取消：按子项执行部分取消并计算退款差额。
- `4.4` 全渠道订单处理池 + 退款审批提交：提交退款审批并进入审核链路。
- `5.1` 团队预订与批量导入 + 批量名单导入弹窗：上传团队名单并做格式与业务校验。
- `5.2` 团队预订与批量导入 + 资源矩阵锁位：按日期与资源维度进行批量锁位。
- `5.3` 团队预订与批量导入 + 团队预订建单：完善团队订单信息并生成可执行团单。
