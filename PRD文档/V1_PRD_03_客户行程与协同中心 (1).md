#### 最小单元补充交互步骤说明

### 2.2.1 3.1.1 行程大纲编辑器

#### 交互步骤（研发可执行）
- 触发条件：订单确认后进入行程编辑器。
- 前置校验：客户信息、服务周期、节点模板完整。
- 按钮动作：生成/更新行程、推送小程序、复制链接、打印。
- 状态反馈：行程版本号递增并记录更新时间。
- 异常提示：节点冲突、推送失败需可重试。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 行程大纲编辑器                                                        [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [客户信息] 客户ID/姓名/套餐/服务周期                                             │
│ [时间轴] 接驳→入住→评估→项目→退房                                                │
├──────────────────────────────────────────────────────────────────────────────┤
│ [编辑区] 节点拖拽/时分秒微调/节点备注                                             │
│ 操作按钮：[生成/更新行程] [推送至小程序] [复制H5短链接] [生成打印版]              │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认发布]                                                      │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.2 3.1.2 行程二次确认核对表

#### 交互步骤（研发可执行）
- 触发条件：客户入住前台核验阶段。
- 前置校验：客户身份匹配、行程存在。
- 按钮动作：勾选确认项、录入偏好、同步下游。
- 状态反馈：核对状态由待核对变更为已核对。
- 异常提示：历史高风险客户需二次确认弹窗。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 行程二次确认核对表                                                    [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 入住日期/客户姓名/核对状态                                             │
│ [预警栏] 迟到/爽约/违约金拒付历史标签                                           │
├──────────────────────────────────────────────────────────────────────────────┤
│ [核对清单] 行程节点勾选 + 风险提示 + 偏好录入                                    │
│ 操作按钮：[核对无误并同步下游] [退回修改]                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认核对]                                                      │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.3 3.2.1 跨部门任务工单

#### 交互步骤（研发可执行）
- 触发条件：行程确认或变更后自动派发工单。
- 前置校验：责任部门与执行人可用。
- 按钮动作：接单、标记完成、异常挂起、任务转派。
- 状态反馈：状态机实时流转并记录处理时长。
- 异常提示：超时未处理自动催办并升级告警。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 跨部门任务工单                                                        [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 责任部门/任务状态/时间区间                                           │
│ [视图切换] 泳道看板 / 明细列表                                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│ [工单区] 任务主题 | 截止时间 | 责任人 | SLA                                      │
│ 操作按钮：[接单] [标记完成] [异常挂起] [任务转派]                               │
│ 完成回执：上传图片/备注 -> 归档                                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认提交]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心状态流转

### 3.1 行程状态

草稿 → 已确认 → 已推送 → 已变更（版本+1）→ 已完成。

### 3.2 协同任务状态

待接单 → 执行中 → 已完成 / 异常挂起 → 转派执行。

---

## 四、术语与字段规范

| 术语 | 说明 |
|------|------|
| 行程二次确认 | 入住前由前台/管家核对关键安排 |
| 协同工单 | 按部门分发的执行任务 |
| 偏好标签 | 忌口/睡眠习惯等服务偏好 |

关键字段：`itinerary_no`、`node_start_time`、`confirm_status`、`dept_owner`、`task_status`。

---

## 五、服务对照表配置

本模块消费服务项定义用于展示行程节点：
- 节点名称、建议时长、执行部门来自服务模板；
- 行程仅做排程，不改写权益主数据。

---

## 六、审核纳入流程（对齐v1.3.1）

纳入后数据用于行程初始化：
- 读取客户基础信息、服务周期、套餐子项；
- 生成首版行程并等待二次确认。

---

## 七、任务中心（对齐v1.1）

本模块任务联动：
- 行程确认/变更触发跨部门任务创建或更新；
- 任务执行结果回写行程节点完成时间。

---

## 八、管理中客户详情（对齐v1.1）

客户详情联动能力：
- 展示行程确认状态与历史异常预警；
- 跳转客户360查看历史履约偏好。

---

## 九、结案流程

结案前校验：
- 所有关键行程节点需完成或明确终止原因；
- 未完成协同任务需强制挂起归档。

---

## 十、任务生成规则配置

规则摘要：
- 行程确认后自动拆分客房/前厅/康养/车队任务；
- 行程节点改期自动同步未完成任务计划时间。

---

## 十一、ERP数据同步与回传

### 11.1 同步输入

- 订单时间、客户信息、服务范围。

### 11.2 回传输出

- 行程确认与执行里程碑供运营与财务查询。
