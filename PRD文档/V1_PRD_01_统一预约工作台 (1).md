#### 最小单元补充交互步骤说明

#### 2.2.A 按钮去向与子页面映射（已讨论锁定）

| 来源按钮 | 去向形态 | 目标页面/容器 | 是否新增设计 | 关键说明 |
|---|---|---|---|---|
| `[帮助]` | 右侧抽屉 | 帮助抽屉（锚点到当前模块FAQ） | 是 | 抽屉内含“功能说明/常见问题/操作视频入口” |
| `[资源筛选]` | 右侧抽屉 | 高级筛选抽屉 | 是 | 支持多条件、保存方案、恢复默认 |
| `[进入处理]` | 子页面 | 异常处理中心（单条Tab） | 是 | 与批量处理共用同一中心页面 |
| `[批量处理]` | 子页面 | 异常处理中心（批量Tab） | 是 | 先预览后执行 |
| `[批量签到]` | 弹层+预览页 | 批量处理模板（签到模式） | 是 | 与批量核销同模板，不同校验 |
| `[批量核销]` | 弹层+预览页 | 批量处理模板（核销模式） | 是 | 增加额度与状态校验 |
| `[收款确认]` | 子页面 | 到店收款确认台 | 是 | 散客/团队统一入口，仅用于资源锁定后的待到店收款确认与归档回传 |
| `[名单导入]` | 子页面 | 导入错误明细页 | 是 | 导入后可逐条修复并重试 |
| `[提交退款审批]` | 右侧抽屉 | 退款审批抽屉 | 是 | 原因+金额+凭证三项必填 |

##### B5. 到店收款确认台（`[收款确认]`）

###### B5-业务决策（已锁定）

| 决策项 | 已确定方案 |
|---|---|
| 入口来源 | 散客建单锁定完成、团队建单锁定完成、订单列表待收款单、到店执行回流 |
| 页面结构 | 三分区（订单区 + 支付区 + 结果区） |
| 支付时点 | 客户到店后，由前台/PMS侧完成收款确认并同步中台 |
| 失败回路 | 留在收银台重试（保留表单数据，可切换支付方式） |

| 成功反馈 | 顶部成功条 + 关键字段高亮 |
| 支付范围 | 仅支持本次订单实收，不支持余额抵扣与分段交付场景 |

## 三、核心状态流转

### 3.1 订单状态流转

草稿 → 预占中 → 已锁定待到店/待收款 → 已支付 → 待履约 → 履约中 → 已完成 / 已违约 / 退款中 / 已关闭 / 已取消。

补充说明：
- 草稿：由`[暂存返回]`或`[存为草稿]`触发，允许继续编辑后再提交。
- 预占中：已发起15分钟预占，等待资源锁定确认。
- 已锁定待到店/待收款：房态/技师/设备资源已锁定成功，等待客户到店并完成收款。
- 已超时关闭：预占倒计时结束仍未完成锁定确认或未进入有效锁定状态时自动流转，并释放库存与资源。
- 已取消：客户未到店前发起取消时触发；中台需校验当前订单节点、释放已锁定房态/技师资源，并执行权益或款项返还。
- 履约中：客户已到店，PMS已完成到店收款或项目执行中。

### 3.2 预占状态流转

预占中（15分钟）→ 已锁定待到店/待收款（资源锁定成功）/ 超时关闭（自动释放）/ 人工取消（立即释放）。

到店收款确认：已锁定待到店/待收款 → 确认到账 → 待履约。

### 3.3 取消与退款流转

客户/业务端发起取消 → 预约中台冻结订单并检查当前流转节点

分支A：未下发康养端/未锁定履约资源
→ 直接更新订单为已取消
→ 退回客户原权益额度或可退金额
→ 同步取消结果至中台与ERP归档

分支B：已下发康养端/已锁定履约资源
→ 下发撤单指令并释放房态/技师/设备资源
→ PMS回传释放结果
→ 若存在已收款项则进入退款审批
→ ERP同步订单作废与财务逆向流水

### 3.4 异常场景与兜底方案

| 异常场景 | 触发节点 | 前端反馈与兜底动作 | 结果状态 |
|---|---|---|---|
| PMS锁房失败 | 发起预占/确认改期 | 弹窗展示失败原因，提供`[重试]`、`[更换房型]`、`[手工录单]`入口 | 预占失败（可重试） |
| 支付回调失败 | 支付完成后状态回写 | 提供`[手动刷新]`，并展示“财务手工核验入口”；系统定时轮询回调 | 支付待确认 |
| 库存被抢占 | 提交预占/提交改期时 | 提示“该时段已被预订”，弹出可用时段/资源智能推荐列表 | 需改选资源 |
| 网络断连 | 任意提交动作中 | 本地缓存草稿，顶部离线横幅提示；恢复网络后自动同步并提示结果 | 草稿保留/自动续传 |

兜底原则：
- 所有失败场景必须“可解释 + 可操作 + 可回退”；
- 所有自动兜底动作必须写入操作日志，便于客服与财务追溯；
- 兜底过程不丢单，优先保证订单与资源状态一致性。

## 五、服务对照表配置

本模块读取服务对照配置，不维护主数据：
- 读取权益项目对应服务项与任务类型；
- 建单时根据服务对照表进行套餐拆分展示；
- 不提供服务对照表新增/编辑能力。

---

## 六、审核纳入流程（对齐v1.3.1）

本模块不承担“审核纳入”主流程，仅消费纳入后的客户与方案数据：
- 已纳入客户可直接在工作台建单/改期；
- 纳入配置中的频次和周期作为建单约束项。

---

## 七、任务中心（对齐v1.1）

本模块与任务中心关系：
- 建单成功后，履约类任务由下游任务中心生成；
- 订单综合列表支持子单核销入口，写回任务完成时间。

---

## 八、管理中客户详情（对齐v1.1）

工作台联动能力：
- 在订单详情中可查看客户基础标签、履约状态；
- 支持跳转客户详情页进行跨模块查看，不在本模块内编辑客户画像。

---

## 九、结案流程

本模块支持“已结案客户”订单只读：
- 禁止新增履约预约；
- 允许查看历史订单、退款与改期记录。

---

## 十、任务生成规则配置

本模块使用统一任务规则：
- 预占超时自动关单；
- 团队批量签到/核销按子单逐条登记；
- 退改调用“阶梯违约金规则”。

## 十一、ERP数据同步与回传

### 11.1 同步输入

- 接收 ERP/渠道同步的订单与客户基础信息；
- 接收 PMS 实时房态变化用于冲突预警。
- 接收 PMS 到店、执行、收款、退房等状态事件用于订单状态推进。

### 11.2 回传输出

- 支付状态与财务确认结果回传至相关财务域；
- 改期、取消、退款、退房结果回传订单中台与下游履约系统；
- 每日财务对账结果与订单作废状态回传 ERP 归档。
