# V1 康养预约调度中心 - 核心功能清单
**版本目标**：保证4.30开业的最小商业与履约闭环 (基于选中方案 M)
**前置依赖**：用于指导原型图（UI/UX）设计

---

## 1. 统一预约工作台 (Unified Booking Workspace)
*调度中心最核心的操作入口，供客服/管家/前台使用。*

- [ ] **多维日历看板**：支持按“天/周/月”切换，左侧Y轴为“房间/康养师/设备”，主体区域展示预约色块（不同状态不同颜色）。
- [ ] **拖拽改约功能**：在日历看板上，支持鼠标拖拽已确认的预约色块到空闲时间段，触发改期二次确认。
- [ ] **手工建单入口**：客服可手工录入散客/协议客订单，包含客户基础信息、选择康养套餐或单项。
- [ ] **套餐自动拆单与两阶段锁位**：选择套餐后，系统自动拆分为“房单”和“项目单”，并展示15分钟倒计时锁位状态。
- [ ] **三池冲突校验提示**：建单时，如果“房态+康养师+设备”存在冲突，强行阻断并推荐最近的3个可用时段。
- [ ] **统一收银与到店收款**：散客/团队订单进入统一收银台，支持全额支付、扫码支付、线下收款确认，并在客户到店执行节点完成收款确认。
- [ ] **退款审批与取消回流**：未到店取消时支持冻结订单、释放资源、计算可退金额，并进入退款审批与驳回回流流程。

## 2. 自营商品与交易中心 (Self-operated Sales & Payment)
*负责康养产品的“上架-售卖-支付”闭环，多端展示。*

- [ ] **商品/套餐配置台**：管理自营康养项目和套餐，包含图文描述、所含子项目、基础售价。
- [ ] **价格日历与库存设置**：设置商品在不同日期的价格，以及每日可约的最大库存量。
- [ ] **收银台/支付路由**：
  - 支持 **全额在线支付** (微信/支付宝)。
  - 支持 **扫码支付 + 线下收款确认**。
  - 支持 **到店执行后确认到账**，并回传预约中台、PMS、ERP财务归档。
- [ ] **差价补收退还面板**：客人改期或换项目产生差价时，生成补收或退款处理，并进入统一审批与财务归档流程。

## 3. 客户行程与体验中心 (Customer Itinerary & Experience)
*面向客户的交付物及内部跨部门的协同纽带。*

- [ ] **客户专属行程单 (C端/H5)**：客户可通过小程序/H5查看完整的可视化时间轴（接驳安排、管家信息、入住指引、康养项目时间表）。
- [ ] **行程二次确认操作**：前台/管家在客户入住时，调出行程单进行二次确认（特别是健康评估时间）。
- [ ] **内部任务联动通知**：行程确认或变更时，自动拆分内部任务列表（通知客房备房、通知康养师准备、通知前台接驾）。

## 4. 渠道接入与监控大盘 (Channel & Integration Management)
*对接外部客源及系统同步健康度的监控。*

- [ ] **渠道防漏单管理台**：查看来自OTA、旧预约系统、直营APP的所有推单记录。
- [ ] **失败重放与幂等处理**：对于解析失败或落单失败的渠道订单，提供手工“一键重试”功能。
- [ ] **接口SLA监控看板**：红绿灯展示与 PMS、康养执行系统、支付网关的连接状态及同步延迟。

## 5. 异常工单与财务对账 (Exceptions & Reconciliation)
*守住资金和客诉底线的安全网。*

- [ ] **异常工单池**：集中展示“PMS锁房失败”、“支付掉单”、“重单”等异常，必须有强制人工处理的闭环流程。
- [ ] **阶梯退改计算器**：客服操作取消时，弹窗根据规则（如距离入住时间>72h, 24h等）自动计算并展示应退金额和违约金。
- [ ] **T+1 三方明细对账表**：每日自动聚合生成报表（调度中心订单 vs PMS账单 vs 支付网关流水）。
- [ ] **差异平账工作台**：高亮标出金额不一致的“长款/短款”订单，提供“标记平账”按钮并强制要求填写备注原因。
