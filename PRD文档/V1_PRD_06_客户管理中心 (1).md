#### 2.2.1 6.1.1 会员全息画像（One-ID）

#### 交互步骤（研发可执行）
- 触发条件：搜索客户或由订单/入住记录进入会员详情。
- 前置校验：One-ID存在；无则按手机号+证件号执行身份合并候选。
- 按钮动作：合并档案、添加偏好标签、查看历史入住与总消费。
- 状态反馈：展示统一主档、跨平台来源、最近同步时间。
- 异常提示：身份冲突（同手机号不同证件）进入人工审核。

│ 会员全息画像（One-ID）                                                   [×]    │

#### 2.2.2 6.1.2 多级会员体系与权益配置

## 三、核心状态流转

会员状态：新建 → 生效 → 冻结/拉黑（可逆）→ 注销。  
积分状态：发放 → 冻结（待核销）→ 核销/过期/回退。  
卡券状态：生成 → 发放 → 已领取 → 已核销/已过期/已作废。  
工单状态：新建 → 已分派 → 处理中 → 已解决 → 已回访关闭。

## 四、术语与字段规范

| 术语 | 说明 |
|------|------|
| One-ID | 跨平台、跨品牌统一会员身份ID |
| RFM模型 | 最近消费(Recency)、消费频率(Frequency)、消费金额(Monetary) |
| 会员贡献率 | 会员收入占总收入比例 |
| 营销ROI | （活动收益 - 活动成本）/ 活动成本 |

关键字段：`one_id`、`member_id`、`member_level`、`member_status`、`point_balance`、`coupon_status`、`stored_value_balance`、`sync_batch_id`、`ticket_id`、`nps_score`。

## 五、规则与任务配置

- 等级规则：支持按间夜量、消费金额、积分阈值单条件或组合条件。
- 权益规则：按等级动态生效，支持生效期、互斥关系、适用渠道配置。
- 积分规则：支持基础规则+活动倍率+优先级覆盖。
- 自动化任务：支持生命周期事件与入住旅程事件双触发。
- 触达频控：按会员/渠道/模板维度配置防骚扰上限。

---

## 六、数据同步与审计策略

- 同步方向：PMS/POS/官网/小程序与CRM双向实时同步。
- 同步机制：事件驱动 + 消息队列 + 幂等键去重。
- 对外接口：提供标准REST API与Webhook回调，支持跨品牌字段映射。
- 审计要求：敏感信息查看、等级变更、资产扣减、工单升级均落审计日志。
- 失败补偿：死信队列 + 自动重试 + 人工补偿台。
