#### 最小单元补充交互步骤说明

### 2.2.1 8.1.1 综合交付列表（参考“其他交付”原型）

#### 交互步骤（研发可执行）
- 触发条件：执行人员进入交付看板。
- 前置校验：预约状态为待交付且执行权限通过。
- 按钮动作：确认交付、查看详细、导出。
- 状态反馈：交付状态更新为已完成并记录执行人。
- 异常提示：高危风险命中时转入阻断流程。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 综合交付列表（参考“其他交付”原型）                                      [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 机构/预约人/日期范围/预约类型/交付状态                                │
│ [列表区] 日期/机构/客户/项目/状态/时间戳/来源标签[老]                           │
├──────────────────────────────────────────────────────────────────────────────┤
│ [确认区] 执行人下拉 + 耗材记录                                                   │
│ 操作按钮：[确认交付] [查看详细] [通用导出] [特定业务导出]                        │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认执行完成]                                                │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.2 8.2.1 财务核销自动路由规则

#### 交互步骤（研发可执行）
- 触发条件：交付完成事件写入。
- 前置校验：凭证字段完整、机构代码有效。
- 按钮动作：系统自动生成凭证并投递MQ。
- 状态反馈：展示待推送/已推送/已核销状态。
- 异常提示：投递失败进入重试队列并告警。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 财务核销自动路由规则                                                    [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [触发区] 8.1.1确认交付完成                                                     │
│ [路由区] MQ -> 按归属地与机构代码推送ERP                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [监控区] 凭证ID/路由目标/推送状态/重试次数                                     │
│ 操作按钮：[查看路由详情] [重试推送]                                             │
├──────────────────────────────────────────────────────────────────────────────┤
│ [关闭] [保存规则] [确认发布]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.3 8.2.2 特殊场景阻断与监控

#### 交互步骤（研发可执行）
- 触发条件：订单带[老]标签或推送失败。
- 前置校验：老客标记来源可信、回调接口可用。
- 按钮动作：阻断ERP推送、回调老中台、手工重推。
- 状态反馈：阻断与回调结果写入监控池。
- 异常提示：回调失败需生成人工排查任务。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 特殊场景阻断与监控                                                      [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [阻断规则] 老客防重扣：带[老]标签 -> 强制阻断ERP推送                         │
│ [回调区] 回调老中台：{'status':'fulfilled','order_id':'xxx'}                │
├──────────────────────────────────────────────────────────────────────────────┤
│ [失败池] ERP/老中台推送失败明细                                              │
│ 操作按钮：[日志排查] [手工重新推送凭证]                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [关闭] [保存阻断规则] [确认处置]                                               │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心状态流转

交付状态：待交付 → 交付中 → 已完成 / 已取消。

核销状态：待推送 → 推送中 → 已核销 / 推送失败（可重试）。

---

## 四、术语与字段规范

| 术语 | 说明 |
|------|------|
| 物理交付 | 现场实际完成服务动作 |
| 核销凭证 | 交付完成后生成的财务确收消息 |
| 老客防重扣 | 带[老]标签订单禁止推ERP |

关键字段：`delivery_id`、`executor_id`、`voucher_id`、`erp_org_code`、`is_legacy_customer`。

---

## 五、服务对照表配置

交付与核销依赖服务对照：
- 服务项与科目映射用于凭证生成；
- 计量类型影响扣减逻辑（次数/重量/实物）。

---

## 六、审核纳入流程（对齐v1.3.1）

纳入后的服务计划为交付看板初始化数据来源。

---

## 七、任务中心（对齐v1.1）

交付完成后回写任务中心：
- 任务状态置已完成并记录执行人；
- 失败重推不重复写任务完成记录。

---

## 八、管理中客户详情（对齐v1.1）

客户详情展示最近交付记录与核销状态，供客服追踪。

---

## 九、结案流程

结案前要求：
- 所有“待核销”凭证处理完毕；
- 老客回调状态同步成功。

---

## 十、任务生成规则配置

本模块触发后置任务：
- 核销失败重试任务；
- 人工排查任务。

---

## 十一、ERP数据同步与回传

### 11.1 同步输入

- 交付执行结果、机构归属参数、客户标记。

### 11.2 回传输出

- 向ERP推送核销凭证；
- 向老中台回调老客履约完成状态。
