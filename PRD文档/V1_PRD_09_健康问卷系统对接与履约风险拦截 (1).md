#### 最小单元补充交互步骤说明

### 2.2.1 9.1.1 外部健康档案拉取接口

#### 交互步骤（研发可执行）
- 触发条件：进入客户详情/预约详情或点击手动拉取。
- 前置校验：外部接口鉴权通过、客户标识有效。
- 按钮动作：手动拉取、查看外部问卷报告详情。
- 状态反馈：同步时间、风险等级、体征高亮更新。
- 异常提示：拉取失败需展示错误码与重试入口。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 外部健康档案拉取接口                                                    [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [入口] 客户360页 / 预约详情页                                                   │
│ [同步模式] 自动静默拉取 + 手动拉取                                               │
├──────────────────────────────────────────────────────────────────────────────┤
│ [展示区] 血压/心率/中医评估等异常体征高亮                                       │
│ 操作按钮：[手动拉取最新问卷结果] [查看外部问卷报告详情]                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ [关闭] [刷新数据] [确认同步]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.2 9.2.1 交付强阻断与授权

#### 交互步骤（研发可执行）
- 触发条件：交付点击确认且命中高危风险标记。
- 前置校验：风险规则版本有效、授权角色存在。
- 按钮动作：终止退改或强行放行（输入授权凭证）。
- 状态反馈：阻断/放行结果写入审计并回写订单状态。
- 异常提示：授权失败禁止放行并提示重试。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 交付强阻断与授权                                                        [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [触发区] 交付列表点击[确认交付]                                               │
│ [判定区] 命中高危标记 -> 红色阻断弹窗                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ [处置区] [终止项目并转售后退改]                                                │
│ [授权区] [风险知情强行放行(需授权)] -> 输入主管/医生密码 -> 审计日志            │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存记录] [确认执行]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心状态流转

问卷数据：待同步 → 已同步 → 已解析。

风险流程：正常 → 预警 → 高危阻断 → 终止退改 / 授权放行。

---

## 四、术语与字段规范

| 术语 | 说明 |
|------|------|
| 高危拦截 | 命中高危规则后禁止常规交付 |
| 强行放行 | 主管/医生授权后继续交付 |
| 风险体征 | 外部问卷返回的异常指标 |

关键字段：`questionnaire_id`、`risk_level`、`risk_flag`、`override_by`、`override_reason`。

---

## 五、服务对照表配置

本模块不维护服务对照，仅在交付动作前做风险拦截判断。

---

## 六、审核纳入流程（对齐v1.3.1）

纳入后首次查看客户或订单时，自动拉取最新问卷摘要用于风险预判。

---

## 七、任务中心（对齐v1.1）

风险事件可触发任务中心：
- 高危阻断生成“风险复核任务”；
- 放行后生成“重点随访任务”。

---

## 八、管理中客户详情（对齐v1.1）

客户详情顶部展示风险标签与关键体征，支持跳转外部报告。

---

## 九、结案流程

高危状态客户结案需附风险处置说明与审批记录。

---

## 十、任务生成规则配置

规则摘要：
- 风险命中时创建阻断事件；
- 强行放行必须双重记录（授权人+时间+原因）。

---

## 十一、ERP数据同步与回传

### 11.1 同步输入

- 外部问卷系统体征与风险标志。

### 11.2 回传输出

- 仅回传履约风险结果摘要，不回传完整问卷明细。

## 十二、待确认事项

| 序号 | 事项 | 说明 | 建议方案 |
|------|------|------|----------|
