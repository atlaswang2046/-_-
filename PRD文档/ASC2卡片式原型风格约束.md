# ASC2卡片式原型风格约束（冻结）

> 适用范围：`PRD文档/V1_PRD_01~09_*.md` 的所有 ASC2 原型块  
> 生效目标：后续任何优化都必须在不破坏已有风格前提下进行

---

## 1) 命令理解（保存）

用户要求已固化为以下不可破坏约束：

1. **保持 Image1 风格**：分层卡片式 ASCII 结构，不可退化为简化清单框。
2. **每个最小单元都要重绘**：不仅是 `2.2.0`，还包括所有 `2.2.1 ~ 2.2.x`。
3. **一标题一图**：每个最小单元标题下必须有对应 ASC2 图。
4. **总览在前，交互在后**：`2.2.0` 作为看板总览；其后保留“最小单元补充交互步骤说明”。
5. **后续优化只允许“视觉密度统一”**，不允许破坏层级、按钮语义和交互结构。

---

## 2) ASC2卡片式标准结构（最小单元）

每个最小单元 ASC2 图建议使用 5 层：

1. **标题栏**：`功能名 + [×]`
2. **信息区**：筛选/基础信息/上下文
3. **内容区**：列表/表格/规则/状态
4. **操作区**：主操作按钮（如 `[操作按钮: ...]`）
5. **底部动作区**：`[取消] 或 [关闭]` + `确认类动作`

---

## 3) 视觉密度统一规则

1. 统一使用 Unicode 线框：`┌┐└┘├┤─│`
2. 区块标签优先使用：`[筛选栏] [列表区] [详情区] [操作区] [操作按钮]`
3. 所有可点击元素统一用中括号表示：`[按钮]`
4. 标题与区块文案不超过两行，避免横向拥挤
5. 底部动作优先采用：`[取消]/[关闭] + [确认xxx]`
6. 不删除已有业务语义，只做排版密度和标签一致性优化

7. 最小单元 ASC2 图必须包含：`标题栏([×]) + 信息/筛选区 + 内容/详情区 + 操作按钮区 + 底部动作区`
8. 顶部总览图（2.2.0）必须包含：`页面工具条 + 概览卡片 + 左右双栏 + 快捷入口`
9. 按钮命名统一：优先使用`[保存草稿]`，禁用`[暂存]`
10. 底部动作优先：`[取消]/[关闭] + [确认xxx]`，确认按钮不得缺失

---

## 4) 变更守则

- 禁止将卡片式结构改回单层清单结构
- 禁止删除 `#### 交互步骤（研发可执行）`
- 禁止调整 `2.2.x` 的菜单编号与顺序
- 允许：统一文案、统一按钮命名、统一区块标签、统一底部动作
