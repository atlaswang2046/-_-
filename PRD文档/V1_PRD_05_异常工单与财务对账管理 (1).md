#### 最小单元补充交互步骤说明

### 2.2.1 5.1.1 PMS / 支付异常池

#### 交互步骤（研发可执行）
- 触发条件：系统检测到支付/PMS状态不一致。
- 前置校验：异常单未关闭且具备处理权限。
- 按钮动作：强行退款关单、替换资源重推、手工录单。
- 状态反馈：异常状态更新并写入处置日志。
- 异常提示：重推超限需主管接管。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ PMS / 支付异常池                                                        [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 异常类型/处理状态/订单号                                               │
│ [列表区] 异常说明 | 影响金额 | 发生时间 | 当前处理人                             │
├──────────────────────────────────────────────────────────────────────────────┤
│ [处置区] 主管强制接管规则 + 重推次数限制                                        │
│ 操作按钮：[强行退款关单] [替换资源重新推送] [强行手工录单]                       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存备注] [确认处置]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.2 5.1.2 售后退单审批管理

#### 交互步骤（研发可执行）
- 触发条件：客服提交退单申请。
- 前置校验：违约金计算完成、审批链路可用。
- 按钮动作：审批通过并退款 / 驳回重申。
- 状态反馈：退款结果回写售后单状态。
- 异常提示：支付网关退款失败时进入异常池。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 售后退单审批管理                                                      [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 售后阶段/订单号/提交人                                               │
│ [单据区] 阶梯违约金计算结果 + 退款凭证                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│ [审批区] 通过后触发第三方原路退款                                              │
│ 操作按钮：[审批通过并退款] [驳回重申]                                          │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认审批]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.3 5.2.1 三方明细对账报表

#### 交互步骤（研发可执行）
- 触发条件：每日T+1自动任务或手工触发。
- 前置校验：三方账单文件完整可读。
- 按钮动作：筛选、导出、重跑对账任务。
- 状态反馈：差异单生成并高亮展示。
- 异常提示：缺账源时标记“对账阻塞”。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 三方明细对账报表                                                        [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [筛选栏] 账期/差异状态（长款/短款/平账）                                      │
│ [报表区] 调度中心订单 vs PMS账单 vs 支付流水                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│ [差异区] 红/黄高亮 + 差异原因字段                                              │
│ 操作按钮：[导出Excel明细账] [手工触发对账任务]                                 │
├──────────────────────────────────────────────────────────────────────────────┤
│ [关闭] [刷新结果] [确认生成差异单]                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2.4 5.2.2 差异单据手工平账台

#### 交互步骤（研发可执行）
- 触发条件：存在未平账差异单。
- 前置校验：平账人权限、必填原因与凭证完整。
- 按钮动作：标记平账、补录关联。
- 状态反馈：差异单状态变更为已平账。
- 异常提示：缺少原因或凭证禁止提交。

```asc2
┌──────────────────────────────────────────────────────────────────────────────┐
│ 差异单据手工平账台                                                      [×]    │
├──────────────────────────────────────────────────────────────────────────────┤
│ [列表区] 未平账差异单：订单号/差异金额/差异类型                                │
│ [详情区] 差异来源 + 历史处理记录                                               │
├──────────────────────────────────────────────────────────────────────────────┤
│ [平账区] 必填原因（线下现金/抹零误差等）+ 附件上传                               │
│ 操作按钮：[标记平账] [手工补录账单关联]                                         │
├──────────────────────────────────────────────────────────────────────────────┤
│ [取消] [保存草稿] [确认平账]                                                    │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心状态流转

异常发现 → 待处理 → 处理中 → 已恢复 / 已退款关单 / 转人工待核实。

对账差异：未平账 → 处理中 → 已平账。

---

## 四、术语与字段规范

| 术语 | 说明 |
|------|------|
| 强行退款关单 | 支付原路退回并关闭本地残单 |
| 长款/短款 | 三方金额不一致导致的差异类型 |
| 平账备注 | 财务手工平账的强制原因字段 |

关键字段：`exception_type`、`ticket_status`、`recon_date`、`diff_amount`、`close_reason`。

---

## 五、服务对照表配置

依赖服务项规则计算退款与违约金：
- 退改时按服务类型+时间窗计算；
- 差异单据保留服务项维度明细。

---

## 六、审核纳入流程（对齐v1.3.1）

纳入流程不在本模块；但纳入信息用于异常定位（客户/方案/归属分部）。

---

## 七、任务中心（对齐v1.1）

异常处理结果联动任务中心：
- 退款关单后关闭相关未执行任务；
- 重推成功后恢复履约任务链路。

---

## 八、管理中客户详情（对齐v1.1）

客户详情页可查看异常与退款历史，不在此模块编辑客户信息。

---

## 九、结案流程

结案前必须满足：
- 无未处理异常工单；
- 对账差异全部进入“已平账”或“豁免已审”。

---

## 十、任务生成规则配置

本模块不生成康养服务任务，仅生成：
- 异常处置任务；
- 财务对账任务与催办提醒。
