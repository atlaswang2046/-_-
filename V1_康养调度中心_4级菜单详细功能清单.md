# V1 康养调度中心 - 详细系统菜单功能清单（4级深度）

> **文档说明**：本清单将 V1 核心商业闭环落地为具体的系统菜单与页面操作，最高细化至 **第4级（具体操作/按钮级别）**。可直接作为开发后台管理系统（B端）以及前端应用（C端/小程序）的路由、权限配置和原型设计目录。

---

## 一、统一预约工作台 (Unified Booking Workspace)
*核心调度人员、前台、管家的高频操作台*

### 1.1 多维调度日历板
- **1.1.1 日历视图看板 (主操作区)**
  - `[筛选]` 多维度资源过滤：
    - 资源类型（下拉单选：按房间SKU / 按康养师排班 / 按医疗设备排班 / 按公共场地）
    - 所属门店（下拉单选：如 海南梅赛尔 / 天津丽滋卡尔）
    - 资源状态（多选：空闲 / 占用 / 锁定 / 维修）
    - 时间范围（日期选择器：默认当天）
  - `[按钮]` 视图控制：[日视图]、[周视图]、[月视图]、[刷新]、[全屏]
  - `[交互]` 预约色块悬浮 (Hover)：弹窗显示“客户姓名、电话(脱敏)、套餐名称、支付状态、履约状态”
  - `[操作]` 拖拽改期：鼠标长按拖拽预约色块至空白时间槽 -> 释放触发后端可用性校验 -> 弹出二次确认框（提示新旧时间对比及可能的违约金差价） -> 点击[确认改期] -> 调用改期API，释放原库存并预占新库存
- **1.1.2 资源冲突预警池**
  - `[列表]` 实时展示当前存在冲突的预约单（如房间被 PMS 强制征用导致系统内双重预订冲突）
  - `[操作]` 智能平替处理：点击[一键智能推荐] -> 调用后端排班算法 -> 弹出模态框展示3个最近的无冲突替代时段 -> 选中一项并点击[确认替换]

### 1.2 订单与预约处理台 (多场景接单中枢)
- **1.2.1 散客建单入口 (直营/到店/电话预订)**
  - `[表单]` 客户基础信息录入（手机号精确匹配画像，姓名、性别、身份证号实时脱敏）
  - `[选择]` 选择康养产品组合（如：康养房SKU + “五维心康管理标准版”套餐 / “高压氧疗”独立项）
  - `[交互]` 购物车模式：支持在单一订单内将多个单项/套餐加入购物车统一结算
  - `[交互]` 联动三池校验：输入时间与项目后，系统静默校验 PMS房态池、康养人效池、设备资源池 是否充足
  - `[按钮]` 流程控制：[存为草稿]、[取消建单]、[发起15分钟预占]、[进入收银]
  - `[操作]` 预占与支付流转：点击[发起15分钟预占] -> 校验必填项 -> 生成草稿单并下发锁房指令至PMS -> 跳转统一收银台 -> 客户到店后点击[生成支付码]或[线下收款确认] -> 点击[确认到账] -> 同步预约中台、PMS 与 ERP 财务归档
  - `[自动化]` 超时释放机制：15分钟倒计时结束后若未完成支付，系统自动将状态流转为“已超时关闭”，并立即释放占用的PMS房态与康养人效设备
- **1.2.2 团队预订建单 (旅行社/商务团队)**
  - `[表单]` 录入团队基础信息：包含“线下合同号/商务协议号、带队导游/销售对接人、预计到店日期”
  - `[操作]` 批量名单导入：点击[下载名单模板] -> 点击[上传Excel] -> 前端解析并调用校验API -> 展示解析明细（高亮标红失败的证件号或冲突房型） -> 点击[确认导入]
  - `[交互]` 集中资源锁位：在资源矩阵图中，框选或填入数量，一键批量预占客房矩阵与大型康养设施（如团体评估室）
  - `[按钮]` 流程控制：[确认生成团单]、[暂存团队预定]、[进入收银]
  - `[操作]` 生成流转：点击[确认生成团单] -> 弹窗二次确认总团费与占用资源 -> 生成团队主单与成员子单 -> 跳转统一收银台 -> 客户到店后完成收款确认 -> 同步预约中台、PMS 与 ERP 财务归档
- **1.2.3 订单综合列表 (多渠道订单处理池)**
  - `[筛选]` 综合查询区：
    - 渠道来源（多选：全部 / OTA美团 / OTA携程 / 线下散客 / 旅行社 / 商务团 / 丽滋卡尔美业渠道）
    - 订单状态（下拉：待支付 / 已支付 / 退款中 / 已关闭 / 已取消）
    - 履约状态（下拉：待签到 / 待履约 / 履约中 / 已完成 / 已违约）
    - 日期区间（精确到时分秒：创建时间区间 / 预约时间区间）
    - 模糊搜索（输入框：订单号 / 姓名 / 手机号）
  - `[视图]` 渠道标签高亮与流转追踪：直观区分带核销码的`[OTA]`与带名册的`[团队]`
  - `[操作]` 树状详情追踪：点击[查看详情] (蓝字) -> 侧滑抽屉拉出追踪树（父订单 -> [关联酒店房单, “脑功能调节”项目子单, “高低氧适应训练”项目子单]）
  - `[操作]` 单项核销确认：选中待履约子单 -> 点击[单项核销] -> 弹窗验证当前登录人权限 -> 调用履约API登记完成时间
  - `[操作]` 团队批量处理：勾选多个团员单 -> 点击[批量签到] 或 [批量核销] -> 执行对应状态变更
  - `[操作]` 售后兜底拆单：在团单中选中个别团员 -> 点击[拆分异常售后单] -> 将该成员从主团剥离，独立计算退改违约金
  - `[操作]` 多场景智能退改：点击[申请退改] -> 联动“退改决策树”，根据距入住时间(>72h/24h等)、散客/团客规则自动计算违约金占比 -> 冻结当前订单并检查履约资源是否已下发 -> 未下发则直接取消并返还权益/可退金额，已下发则先撤单释放资源 -> 点击[提交退款审批]
  - `[操作]` 独立部分取消：支持在一笔包含客房与康养项目的综合单中，单独取消客房（解绑房态）或单独取消康养项目（释放时段）

## 二、自营商品与交易中心 (Product & Sales)
*维护康养医院/酒店可售卖的产品库与价格策略*

### 2.1 商品库管理
- **2.1.1 康养单项管理**
  - `[筛选]` 商品查询：状态(上架/下架)、分类(设备类/产品类/交付类/服务类/餐饮类)、商品名称搜索
  - `[按钮]` 商品管控：[新增单项目]、[批量上架]、[批量下架]、[导出商品]
  - `[表单]` 基础信息库：录入项目名称（如“高压氧疗”）、图文介绍、基准时长、所需设备类型、人员资质要求
  - `[操作]` 核销规则配置：点击[配置核销规则] -> 弹出规则表单 -> 选择“按次/按周期/按节点” -> [保存规则]
- **2.1.2 康养套餐配置**
  - `[按钮]` 套餐管控：[新建套餐]、[草稿保存]、[上架套餐]
  - `[交互]` 组合打包：在双栏穿梭框中勾选内部单项（客房SKU + 如“高压氧疗”单项 + 如“药膳食疗课程”单项），自动核算组合成本价
  - `[设置]` 售卖规则配置：可配置该套餐的“成团人数下限（针对团队商品）”及“允许售卖的渠道（定向发布至OTA或企业直营）”
  - `[操作]` 自动拆单规则配置：点击[配置拆单策略] -> 勾选哪些子项抛给PMS建账，哪些留给康养执行系统 -> [保存提交]

### 2.2 价格日历与库存调度
- **2.2.1 每日价格看板**
  - `[筛选]` 月份切换控件、商品/套餐名称精确搜索
  - `[按钮]` [导出价格表]、[批量设置价格]
  - `[操作]` 批量调价策略：点击[批量设置价格] -> 弹窗内框选日期范围 -> 填入基础价、周末加价、法定节假日加价 -> [保存并生效]
- **2.2.2 容量与库存管理**
  - `[筛选]` 资源类别、日期范围
  - `[按钮]` [批量修改库存]、[特殊锁仓拦截]
  - `[操作]` 强行锁仓停售：选中特定日期与商品 -> 点击[特殊锁仓拦截] -> 填写停售原因(如设备检修) -> 阻断外部OTA及内部正常建单

### 2.3 支付与收银台配置
- **2.3.1 支付路由规则**
  - `[筛选]` 支付渠道状态(开启/关闭)
  - `[操作]` 启停支付方式：通过全局 Switch 开关直接控制（微信、支付宝、扫码支付、线下收款确认等），点击即时生效
- **2.3.2 线下/对公转账审核**
  - `[列表]` 待审核流水池：展示财务需核对的外部转账单
  - `[按钮]` [认领到账]、[驳回打回]
  - `[操作]` 确收到账点亮订单：点击[认领到账] -> 弹窗要求输入财务系统的真实对公流水号与核实金额 -> 点击[确认认领] -> 触发业务单状态流转为“已支付”
- **2.3.3 差价补收退还面板**
  - `[操作]` 生成补收款：输入差额 -> 点击[生成补款码] -> 弹出含金额的动态二维码供客户扫码
  - `[操作]` 提交退还流：输入应退差价 -> 点击[提交退费审批] -> 填写备注并上传凭证 -> 流转至运营初审与财务复审

## 三、客户行程单与协同中心 (Customer Itinerary)
*面向C端的服务触点及内部任务分发*

### 3.1 客户行程单管理
- **3.1.1 行程大纲编辑器**
  - `[视图]` 动态时间轴：横向或纵向展示客户从入住到离店的全生命周期（接驳 -> 入住 -> 评估 -> 康养项目如“深睡舱体验” -> 退房）
  - `[按钮]` [生成/更新行程]、[推送至小程序]、[复制H5短链接]、[生成纸质打印版(用于导出或手环扫码校验)]
  - `[操作]` 微调节点：拖拽时间轴节点，或点击[编辑时间]按钮精修时分秒 -> 触发表单修改保存
- **3.1.2 行程二次确认核对表**
  - `[筛选]` 检索客户：入住日期、客户姓名搜索、核对状态（待核对/已核对）
  - `[视图]` 历史异常预警：若该客户历史上有过“迟到、爽约、违约金拒付”记录，在核对表顶部展示黄色预警标签提醒前台加强确认
  - `[操作]` 前台入住核验：输入客户信息调出该单 -> 业务员勾选所有“已口头确认”的项目 -> 录入特殊偏好（如忌口、睡眠习惯） -> 点击[核对无误并同步下游]，完成履约前二次确认

### 3.2 内部协同任务大盘
- **3.2.1 跨部门任务工单**
  - `[筛选]` 协同视图过滤：责任部门（客房部/前厅部/康养部/车队）、任务状态（待接单/执行中/已完成/异常挂起）、时间区间
  - `[视图]` 多维度展示：支持切换为泳道看板或明细列表（如客房部展示“提前备房任务及偏好”、车队展示“按行程接驳派车”）
  - `[按钮]` 状态流转操作：[接单]、[标记完成]、[异常挂起]、[任务转派]
  - `[操作]` 任务执行登记：员工点击[接单] -> 任务进入执行中 -> 物理执行完成后点击[标记完成] -> 可选上传实地照片或文本备注 -> 任务归档闭环

## 四、渠道接入与系统监控大盘 (Channel & Integration)
*保障多来源客单不漏、不错，系统接口健康*

### 4.1 渠道网关管理台
- **4.1.1 渠道参数配置**
  - `[筛选]` 渠道网关状态
  - `[操作]` 外部来源启停：点击列表对应渠道（OTA渠道、**丽滋卡尔原预约中台**、小程序直连）的 Switch 开关
  - `[按钮]` [配置映射表(SKU Mapping)]
  - `[操作]` 映射配置：点击[配置映射表] -> 进入左右双栏界面 -> 左侧为外部渠道SKU，右侧下拉选择本地对应的产品SKU -> 点击[保存映射关系]
- **4.1.2 渠道落单监控**
  - `[筛选]` 监控过滤器：渠道来源、解析状态（成功/失败/重试中）、报文接收时间
  - `[操作]` 查阅原始数据：点击[查看原始报文] -> 弹出对话框以高亮格式展示接收到的 JSON/XML 原始字符串
  - `[按钮]` 异常兜底：[单条失败重放]、[一键批量重推建单] -> 再次将报文压入解析队列验证

### 4.2 接口 SLA 与健康度监控
- **4.2.1 外部系统拓扑图**
  - `[筛选]` 监控时间窗口：最近1小时、最近24小时
  - `[视图]` 动态连通拓扑：展示 调度中心与 PMS、旧中台、外部健康问卷系统等连通性节点
  - `[提示]` 红绿灯状态告警：若连续N次接口心跳超时，节点标红并弹出“接口超时/熔断报警”
  - `[自动化告警]` 双渠道推送：熔断发生时，通过站内信和[钉钉消息]实时推送给运营负责人与技术值班人员

## 五、异常工单与财务对账管理 (Exceptions & Finance)
*最后一道防线：钱账一致，退改合规*

### 5.1 异常兜底工单池
- **5.1.1 PMS / 支付异常池**
  - `[筛选]` 异常池条件：异常类型（支付成功但PMS锁房失败 / 库存被抢占 / 网络断连）、处理状态
  - `[按钮]` 应对措施：[强行退款关单]、[替换资源重新推送]
  - `[操作]` 强行退款：点击[强行退款关单] -> 系统自动调用支付网关原路退回并关闭本地及PMS未完成的残表 -> 记录强行关单日志
  - `[操作]` 换房重推：点击[替换资源重新推送] -> 弹窗选择当前可用的新房型/资源 -> 调用 PMS 重推接口
  - `[操作]` 手工兜底录单：当外部接口（PMS/康养系统）严重超时熔断时，点击[强行手工录单] -> 生成状态为“待核实”的订单，用于事后人工比对平账
  - `[强制接管]` 客服强力介入：重推超过限制次数后，允许主管权限进行强制干预（如强制标记成功或强制关单退款）
- **5.1.2 售后退单审批管理**
  - `[筛选]` 售后阶段（待审批/审批中/已退款/已驳回）、订单号、提交人
  - `[操作]` 审核与执行退款：查阅前端抛出的“阶梯退费违约金计算结果” -> 财务或主管点击[审批通过并退款] -> 触发系统调用第三方支付原路退回；或点击[驳回重申]

### 5.2 T+1 财务对账台
- **5.2.1 三方明细对账报表**
  - `[筛选]` 财务报表过滤：账期（某年某月某日）、差异状态（存在长款 / 存在短款 / 无差异平账）
  - `[按钮]` [导出Excel明细账]、[手工触发对账任务]
  - `[交互]` 自动化差异高亮：每日凌晨自动化执行聚合后，在列表中对存在金额不一致（调度中心订单 vs PMS账单 vs 支付网关流水）的记录以显眼的红/黄色背景高亮
- **5.2.2 差异单据手工平账台**
  - `[筛选]` 仅查看未平账差异单
  - `[按钮]` [标记平账]、[手工补录账单关联]
  - `[操作]` 强制备注平账：勾选差异单据 -> 点击[标记平账] -> 弹出原因录入框（必填，如：线下补退现金、抹零误差等） -> 点击[确认提交标记]

## 六、客户管理中心 (Customer Management)
*沉淀客户资产，提供权益核销与个性化服务的基础*

### 6.1 客户档案与 360° 画像详情
- **6.1.1 客户综合列表**
  - `[筛选]` 多条件精细检索：姓名、手机号、会员等级、客户身份(普通/VIP等)、客户来源、风险状态
  - `[按钮]` [新建客户信息]、[编辑基本资料]、[标记黑名单/高风险]、[导出客户名册]
- **6.1.2 全局客户详情顶栏 (客户基本信息)**
  - `[视图]` 核心标识栏：展示唯一ID、姓名、性别、年龄、客户等级及各类归属线（大区-门店-管家）

## 七、系统基础管理 (System Administration)
*支撑系统安全运转的基础 RBAC 权限与全局配置*

### 7.1 组织架构与权限 (RBAC)
- **7.1.1 部门与组织架构管理**
  - `[视图]` 树状层级：多级维护树（集团-BU-区域-门店-科室）
  - `[按钮]` [同步钉钉组织架构]、[新增子部门]、[编辑]、[停用]
  - `[操作]` 一键钉钉同步：点击[同步钉钉组织架构] -> 调用钉钉API -> 对比本地树差异 -> 自动补全新职员与部门节点 -> 刷新视图
- **7.1.2 角色与权限引擎**
  - `[表单]` 角色模型创建：录入角色名称、角色编码、数据权限范围下拉（本部门/本门店/全集团）
  - `[操作]` 精准菜单赋权：以树状复选框形式呈现系统的所有路由及本清单中的所有按钮（细化到指令级），勾选后点击[保存授权]
  - `[操作]` 反向赋权(从角色配用户)：在角色列表点击[关联用户] -> 弹窗左侧显示部门树，右侧展示员工列表 -> 批量勾选员工 -> 点击[确认关联]
- **7.1.3 员工(用户)账号管理**
  - `[筛选]` 综合查询：账号、姓名、所属部门下拉树、账号状态（正常/冻结）
  - `[表单]` 编辑用户(正向赋权)：维护基础信息（账号、工号、职务、手机、邮箱），提供“角色分配多选框”直接给员工挂载多个角色
  - `[按钮]` 账号管控：[启用/停用]、[重置默认密码]、[导出人员名单]

### 7.2 系统全局配置
- **7.2.1 数据字典维护**
  - `[筛选]` 字典分类层级树查询
  - `[操作]` 增删改查：针对证件类型、常见取消原因等，点击[新增键值对] -> 填入标签与字典值 -> [保存发布]
- **7.2.2 消息通知模板配置**
  - `[表单]` 模板配置器：选取触发节点下拉（如：预定成功、改期提醒），填入含动态变量`${xxx}`的站内信或短信文案 -> 点击[保存模板]
- **7.2.3 安全与审计日志**
  - `[筛选]` 日志查询：操作人姓名、操作模块、操作类型(增/删/敏感查询)、时间范围
  - `[视图]` 不可篡改列表：展示IP来源、动作详情、入参及影响的业务单号

## 八、康养服务交付与财务核销中心 (Service Delivery & Verification)
*承接预约单的物理交付动作，并作为数据枢纽向对应组织ERP下发财务核销凭证*

### 8.1 治疗/项目交付看板 (物理交付动作台)
- **8.1.1 综合交付列表 (参考"其他交付"原型)**
  - `[筛选]` 顶部核心查询栏：
    - 机构维度（勾选框：仅看本级店铺预约）
    - 预约人信息（精确文本框：姓名/手机号/证件号）
    - 日期范围（日期区间组件：按预约日期查找）
    - 预约类型（下拉选项：设备调理/服务项目/套餐等）
    - 交付状态（下拉选项：待交付 / 交付中 / 已完成 / 已取消）
  - `[视图]` 多维交付数据表：直观呈现日期、机构(如梅赛尔医院)、客户身份、具体项目(如“MK氧疗”)、状态时间戳等
  - `[标签]` 老客户身份防重标识：在来源是“丽滋卡尔中台”的记录上，高亮打上红底白字的 `[老]` 标签
  - `[按钮]` 业务流转：[确认交付] (主功能绿按钮)、[查看详细] (辅功能蓝按钮)、[通用导出]、[特定业务导出(如双美)]
  - `[操作]` 确认物理交付：点击[确认交付] -> 弹出操作确认窗 -> 下拉选择实际执行人(主治医生/康养师)并可选填写耗材耗损 -> 点击最终[确认执行完成] -> 更新状态为“治疗完成”
  - `[操作]` 查看详情穿透：点击[查看详细] -> 弹出宽屏侧滑抽屉或新标签页，展现该订单业务追踪生命周期与关联画像记录

### 8.2 ERP财务核销流转引擎 (跨系统财务同步)
- **8.2.1 财务核销自动路由规则**
  - `[自动化]` 异步消息队列：当 8.1.1 完成上述“确认物理交付”操作后，系统生成标准的“待财务核销凭证（包含：金额、使用的资产科目、机构代码）”，并推入 MQ 队列
  - `[路由分发]` 系统依据归属地参数（如海南/天津）和发生地机构代码，动态推送到对应子组织的底层 ERP 系统（如金蝶、用友）执行最终的确收扣减
- **8.2.2 特殊场景阻断与监控**
  - `[自动化]` 老客防重扣阻断：当解析到该笔已完成交付的订单带有 `[老]` 标签时，拦截层**强制阻断**将核销凭证发往ERP。转而调用特殊的回调 API，向老系统（丽滋卡尔原中台）回传 `{'status': 'fulfilled', 'order_id': 'xxx'}`，由老系统自行完成资产账务处理
  - `[列表]` 同步失败监控池：查询筛选因网络或脏数据导致推送 ERP/老中台 失败的队列明细
  - `[按钮]` [日志排查]、[手工重新推送凭证]

## 九、健康问卷系统对接与履约安全拦截 (Health Questionnaire Integration)
*说明：健康问卷评估由外部独立项目/系统承接，调度中心仅做核心数据拉取与交付前的安全校验，不内置复杂问卷与算法，保障开业进度。*

### 9.1 健康评估数据同步与展示
- **9.1.1 外部健康档案拉取接口**
  - `[按钮]` [手动拉取最新问卷结果]、[查看外部问卷报告详情]
  - `[操作]` 数据联调同步：在查看某预约单或进入客户360画像时，后端自动通过API静默获取最新填写的问卷体征与中医评估数据
  - `[视图]` 风险体征展示：在面板顶部醒目位置高亮显示外部接口返回的超标数值（如“血压偏高(160/100)”、“心率异常”）

### 9.2 高危风险熔断机制 (履约前拦截)
- **9.2.1 交付强阻断与授权**
  - `[交互]` 红色预警拦截：操作人员在【8.1.1 交付列表】中点击[确认交付]时，若系统检测到该单客户的健康问卷存在“高危拦截”标志，原定交付确认弹窗将被红色警示窗替代，阻断常规流程
  - `[按钮]` [终止项目并转售后退改]、[风险知情强行放行(需授权)]
  - `[操作]` 强行放行提权：点击[强行放行] -> 弹出主管/医生密码输入框 -> 验证权限与密码通过后 -> 记录特批审计日志并完成交付
