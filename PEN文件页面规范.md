# PEN文件页面规范

> 适用文件：`原型图/01统一预约工作台V1.0.pen`
> 
> 执行要求：以后每次**新建页面**或**调整页面**前，先阅读并遵循本规范。

## 1. 页面分级与布局规则

### 1.1 页面命名分级
1. `PX.0` 为一级页面。
2. `PX.X` 为二级页面。
3. `PX.X.X` 为三级页面。

### 1.2 画布排布规则
1. 一级页面：以 `P1.0` 为基准，纵向对齐排列。
2. 二级页面：在对应一级页面右侧横向展开。
3. 三级页面：在对应父级页面正下方排列。
4. 页面间距保持一致，避免交叠。

### 1.3 页面尺寸规则（当前生效）
1. 页面默认最小尺寸：`1920 x 1080`。
2. 页面内容超出默认宽度时，页面宽度自动向右扩展。
3. 页面内容超出默认高度时，页面高度自动向下扩展。
4. 禁止内容被画布裁切；调整后需做布局问题扫描（`problemsOnly=true`）。

## 2. 侧边栏规则

### 2.1 结构与位置
1. 标准侧边栏节点名：`Nav/UnifiedV2`。
2. 侧边栏必须左贴边：在其父级 Body 容器中 `x=0`。
3. 侧边栏标准宽度：`200`（特殊页面可按已存在样式保留，但需统一同一组页面）。

### 2.2 高度与联动
1. 侧边栏高度必须设置为：`height: "fill_container"`。
2. 其父级 `BodyLayout`/主内容容器必须可撑满页面剩余高度（通常为 `fill_container`），确保侧边栏可延伸到页面底部。
3. 不允许因为父容器固定高度或额外 padding/gap 导致侧边栏“看起来没到底”或“未左对齐”。

### 2.3 视觉规范
1. 一级页面侧边栏为深色方案：主背景 `#0F2747`，右侧分割线 `#1E3C66`。
2. 其他页面默认沿用所属流程页面的侧边栏样式，优先与同流程页面一致。

## 3. Element Plus 页面组件规则

## 3.1 页面骨架（优先复用）
1. `TopNav`（顶部导航）：常用高度 `60`。
2. `BodyLayout`（主体容器）：承载 `Nav/UnifiedV2 + MainArea`。
3. `MainArea`（主内容区）：常见 `padding: [16,20]`，`gap: 12`。
4. 页面优先使用已有可复用组件，不重复造轮子。

### 3.2 组件风格基线
1. 卡片/面板：白底为主（`#FFFFFF`），常用圆角 `4/6/8`。
2. 描边：优先使用浅灰线（如 `#EBEEF5` / `#DCDFE6` / `#E5EAF3`）。
3. 表格、筛选区、工具栏风格在同页面内保持一致。
4. 禁止在同一页面混用冲突的间距、边框、圆角规范。

### 3.3 组件使用原则
1. 优先使用已存在的 Element Plus 语义组件与变体（如 Button/Input/Select/Table/Dialog/Drawer/Pagination 等）。
2. 同功能组件在不同页面保持同名结构和交互层级（例如：筛选区、列表区、弹窗头/体/尾）。
3. 新页面优先从同类型页面复制骨架，再改业务内容，不从零拼装。

## 4. Note 标签规则

### 4.1 位置规则
1. 每个页面应有对应 Note。
2. Note 放在对应页面右侧，不得遮挡页面主体。
3. 推荐放置方式：`note.x = page.x + page.width + 20`，`note.y = page.y`。

### 4.2 尺寸与命名
1. Note 常用尺寸：`250 x 260`。
2. 命名规范：`note{页码}`，例如 `note3_1`、`note4_3_2`。

### 4.3 内容模板
1. 内容建议统一为 6 段：
   - 页面标题
   - 🔵 业务逻辑
   - 🟢 操作流程
   - 🟠 必填项
   - 🔴 校验规则
   - 🟡 约束项
2. 文案应可直接指导产品、设计、开发、测试理解页面意图。

## 5. 新建/调整页面执行清单（必须）

1. 先确认页面层级（一级/二级/三级）与画布摆放位置。
2. 套用标准骨架：`TopNav + BodyLayout + Nav/UnifiedV2 + MainArea`。
3. 检查页面最小尺寸 `1920x1080`，超出内容自动扩展宽/高。
4. 检查侧边栏左贴边、`fill_container`、并可延展到底部。
5. 检查组件风格一致性（间距、圆角、描边、表格/弹窗结构）。
6. 在页面右侧新增/更新 Note（命名、位置、内容模板一致）。
7. 最后执行布局问题扫描，确保无裁切、无重叠、无异常。

---

## 6. 维护说明

1. 本规范是当前 `.pen` 文件的执行基线。
2. 后续若规则调整，先更新本文档，再批量改页面。
3. 新增页面或大范围改版，必须先对照本规范进行一致性检查。
