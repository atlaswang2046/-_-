# V1_UI修改执行清单 (V1 UI Modification Checklist)

> **文档说明**：本清单基于《V1_康养调度中心_4级菜单详细功能清单》与原型对比分析整理，旨在明确 UI 原型需补充与优化的具体细节。

---

## 1. 导航结构集成 (Navigation Integration)
**目标**：确立“统一预约工作台”在系统中的核心入口地位。

- [ ] **新增一级导航入口**：
    - 在系统侧边栏/顶栏新增一级菜单：`统一预约工作台` (Unified Booking Workspace)。
    - **图标建议**：日历或调度中心样式图标。
- [ ] **配置二级子菜单**：
    - `1.1 多维调度日历板`：资源排期视图。
    - `1.2 订单与预约处理台`：包含散客/团队建单入口及订单综合列表。
- [ ] **跨页面跳转逻辑优化**：
    - **日历板 -> 建单**：在日历板空白时间槽增加“点击”事件，跳转至“散客建单”表单，并自动带入选中的日期与时段。
    - **日历板 -> 详情**：点击预约色块触发右侧抽屉（Drawer）或弹窗（Modal），展示订单简要信息并提供“查看详情”链接。

---

## 2. P5 团队预订流程优化 (Team Booking Flow)
**目标**：在正式生成团单前增加名单预览与核对环节，防止误操作。

- [ ] **新增“预览名册”步骤**：
    - 在团队预订建单页（1.2.2），点击 `[确认生成团单]` 按钮后，不再直接提交，而是弹出 **【名单预览核对】** 弹窗。
- [ ] **名单预览弹窗 UI 设计要求**：
    - **表头信息**：展示团队名称、预计到店日期、合同号、总人数。
    - **列表展示**：表格形式展示解析后的 Excel 名单（姓名、身份证、房型需求、康养项目）。
    - **异常标记**：对于身份证校验失败或资源冲突的行，需使用 **红底白字** 或 **感叹号图标** 高亮。
- [ ] **按钮交互调整**：
    - 弹窗底部增加两个按钮：`[返回修改]` 和 `[确认并支付/预占]`。
    - 只有在名单通过初步校验（无逻辑冲突）后，`[确认]` 按钮才可点击。

---

## 3. P4/P3 订单列表状态扩展 (Order List Statuses)
**目标**：完善订单生命周期，覆盖暂存、超时及部分执行场景。

- [ ] **更新订单状态标签 (Status Tags)**：
    - **草稿 (Draft)**：
        - **场景**：用户点击 `[存为草稿]` 或 `[暂存返回]`。
        - **UI 表现**：灰色背景标签。
        - **操作权限**：列表操作列增加 `[继续编辑]` 按钮。
    - **已超时 (Timeout)**：
        - **场景**：15分钟预占支付倒计时结束。
        - **UI 表现**：淡紫色或淡棕色背景标签。
        - **提示信息**：列表项需显示“支付超时，资源已释放”的文本提示。
    - **部分履约 (Partial Fulfillment)**：
        - **场景**：订单包含多个项目，部分已核销。
        - **UI 表现**：蓝色渐变或蓝色背景标签。
        - **详情联动**：点击详情后，需清晰区分“已完成”与“待执行”的子单项。
- [ ] **操作列按钮适配**：
    - 针对 `草稿` 状态：操作列显示 `[编辑]`、`[删除]`。
    - 针对 `待支付` 状态：增加 `[立即支付]` 及 `[15:00]` 倒计时动态显示。

---

## 4. UI 细节修正建议 (Actionable Steps for Designer)

1.  **颜色规范**：确保 `[OTA]` 渠道标签与 `[团队]` 渠道标签在颜色上有明显区分（如：橙色 vs 蓝色）。
2.  **气泡提示**：在日历板的预约色块上增加 Hover 气泡，展示脱敏后的客户姓名和联系方式。
3.  **空状态处理**：设计日历板在无排班/无资源时的“空状态”占位图。
4.  **移动端适配**：考虑到管家可能使用手持设备，交付列表（8.1.1）的 `[确认交付]` 按钮需加大点击区域。
