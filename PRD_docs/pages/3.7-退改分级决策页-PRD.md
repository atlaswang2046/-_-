# 3.7 退改分级决策页 PRD

## Purpose
根据核销与交付状态、时间窗口与规则引擎输出自动或人工退改决策，减少人工干预并保障合规。

## Target Users
- 客服、财务、运营、审批人

## Scope
- 策略判定、自动处理、人工审批与执行记录

## Inputs
- 退改申请、订单核销/交付状态、策略规则集

## Outputs
- refund_change_request、refund_change_decision、操作日志

## Flow
1. 申请入库 -> 2. 策略判定 -> 3a. 自动退改 -> 3b. 人工审批 -> 4. 执行并回写结果

## UI (ASC2)
- 申请信息、规则判定面板、执行区（自动/人工）

## Exceptions
- 跨系统冲突强制人工审批、审批超时自动升级

## Acceptance Criteria
- 自动处理命中率提升到合规界限（由业务定义）
