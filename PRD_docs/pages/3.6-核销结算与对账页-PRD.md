# 3.6 核销结算与对账页 PRD

## Purpose
完成交付后订单核销、分账与日终/月终对账，发现异常时触发财务工单与人工复核。

## Target Users
- 财务、运营、对账人员

## Scope
- 全额/部分核销、分账机制、差异池、报表导出与工单触发

## Inputs
- 核销回执、支付流水、分账规则、对账文件

## Outputs
- verification_record、settlement_statement、财务工单

## Flow
1. 交付确认触发核销 -> 2. 执行分账 -> 3. 日终/月终对账 -> 4. 异常工单生成

## UI (ASC2)
- 筛选区、对账列表、差异池卡、导出/发起分账按钮

## Exceptions
- 分账失败自动重试3次后生成工单，差异超阈值冻结结算

## Acceptance Criteria
- 日终对账通过率达到95%（异常工单覆盖）
