# 3.1 多场景预约受理工作台 PRD

## Purpose
统一接收并标准化多渠道预约请求，提供筛选、合单/驳回、补录与路由能力，作为预约调度与后续编排的入口。

## Target Users
- 预约受理人员（运营）
- 销售、管家、前台

## Scope
- 处理来自 8 类渠道及运营后台的预约请求
- 支持创建、导入、查询、补录、合单、路由操作

## Inputs
- 渠道来源、客户信息、预约意图、时间窗、权益与支付线索

## Outputs
- 标准化预约请求单（channel_booking_request）

## Flow (High level)
1. 多渠道请求入队列 -> 2. 去重/合单判定 -> 3. 字段完整性校验 -> 4. 资源可约性初步判断 -> 5. 生成标准请求单 -> 6. 路由到资源校验/下单/人工补录

## UI (ASC2)
参见原PRD中 3.1 ASC2 页面设计（保留筛选区、请求列表与操作列）

## Data / DB refs
- channel_booking_request 表

## Exceptions & Error Handling
- 重复请求提示合单/驳回
- 字段缺失进入补录队列并标记风险

## Acceptance Criteria
- 支持所有 8 类渠道入单
- 重复判定准确率>=95%（规则化）
- 补录流程可追踪并生成工单
