# 3.3 资源校验与下单页 PRD

## Purpose
联合校验房型、医疗号源、康养项目与接驳车辆，完成支付/权益核销并生成订单与预约单。

## Target Users
- 客户/前台/管家/销售

## Scope
- 资源可约性判断、支付方式、权益抵扣、订单预览与确认下单

## Inputs
- 房态、号源、项目库存、车辆可用性、权益余额

## Outputs
- booking_order、booking_item、resource_lock_record

## Flow
1. 资源校验 -> 2. 支付与权益计算 -> 3. 生成订单与预约单 -> 4. 同步下游系统

## UI (ASC2)
- 资源区、权益区、支付区、订单预览、确认下单按钮

## Exceptions
- 资源不可约时展示备选时段；支付失败提供重试或转人工

## Acceptance Criteria
- 资源校验准确、下单链路无丢单、支付与权益核销日志完整
