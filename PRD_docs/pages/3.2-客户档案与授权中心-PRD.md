# 3.2 客户档案与授权中心 PRD

## Purpose
建立与维护客户主档、电子授权与医康数据同步，为后续资源校验、权限控制与合规审计提供基础数据。

## Target Users
- 客户服务、管家、运营、审计

## Scope
- 客户识别（新/老客）、电子授权、数据同步日志、审计检索

## Inputs
- 客户个人信息、授权签署、外部系统（CRM/HIS/PMS）回执

## Outputs
- customer_profile、customer_authorization、同步回执日志

## Flow
1. 客户识别 -> 2. 建档/合并（去重） -> 3. 发起/记录授权 -> 4. 同步外部数据/记录日志

## UI (ASC2)
- 客户信息卡、授权状态卡、数据同步日志与审计检索

## Exceptions
- 同步失败触发运营告警（连续3次）

## Acceptance Criteria
- 客户识别准确率>=98%
- 授权签署链路可审计并保存回执
