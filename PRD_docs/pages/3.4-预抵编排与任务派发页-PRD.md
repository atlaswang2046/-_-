# 3.4 预抵编排与任务派发页 PRD

## Purpose
在客户到店前完成预抵沟通、问卷回收、行程单生成与内部任务派发，确保交付各方按计划执行。

## Target Users
- 管家、前台、礼宾、客房、餐饮、健康评估师

## Scope
- 管家指派、问卷发送与收集、行程单生成、部门任务看板与催办

## Inputs
- 预约单、客户偏好、问卷结果、SLA配置

## Outputs
- dispatch_task、service_delivery_record、pre_arrival_itinerary

## Flow
1. 管家指派 -> 2. 问卷推送与回收 -> 3. 生成行程单 -> 4. 自动派单到相关部门 -> 5. 任务追踪与催办

## UI (ASC2)
- 客户卡、问卷区、行程轴、部门任务看板、任务列表

## Exceptions
- 管家首次联系超时自动升级，任务到期自动催办

## Acceptance Criteria
- 行程单正确生成并下发任务，SLA告警准确触发
